# ADR 0009: Contracts-driven Types (Generated Only)

- Status: Accepted
- Date: Unknown
- Scope: Contracts-driven Types (Generated Only)

> Source of truth: code and tests override ADR text. See ADR 0052 when documentation and implementation diverge.

## 背景

ZEN70 是多语言多服务体系（Python/FastAPI 网关、探针/worker、前端 TypeScript）。在长期演进中，**手写 DTO** 极易出现字段漂移、可选性不一致、错误码结构不一致，导致：

- 前端渲染无法依赖稳定契约（尤其是 `code/message/recovery_hint/details`）
- 微服务之间传输对象难以保证兼容
- 回归成本高，问题难追踪

## 决策

- 引入 `contracts/` 目录作为**契约唯一真理源**：
  - OpenAPI：网关 FastAPI 自动导出到 `contracts/openapi/zen70-gateway.openapi.json`
  - 未来可扩展：Protobuf（事件流/内部 RPC）
- 禁止在多个语言/服务内**重复手写数据结构**：
  - 前端 TypeScript 类型与 API 客户端：由 OpenAPI 自动生成
  - Go/Python 共享结构：由契约自动生成
- 在生成文件与元数据中写入**机器可读溯源信息**：
  - `@generated by ZEN70-AI-Agent`
  - `@architecture-version: V2.0`
  - `@compliance: strict-sre-ruleset`

## 后果

- 优点：
  - 契约一致，避免字段漂移与“半更新”
  - 前端可稳定按统一错误契约动态渲染提示/操作建议
  - CI 可以加入“契约生成后 diff 必须为空”的门禁
- 代价：
  - 需要维护契约导出与代码生成工具链（Node/Python）
  - 初期会暴露历史遗留的类型/协议不一致问题，需要逐步整改
